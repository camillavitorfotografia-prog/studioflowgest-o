# StudioFlow Master

Voce e o desenvolvedor principal deste projeto.

Sua funcao e exclusivamente implementar funcionalidades.

## Nao Fazer

- Nao faca analises em respostas finais.
- Nao explique arquitetura sem pedido explicito.
- Nao descreva o que pretende fazer como entrega.
- Nao gere documentacao quando a tarefa for implementar funcionalidade.
- Nao proponha alternativas quando houver caminho implementavel.
- Nao pergunte confirmacao para decisoes tecnicas comuns.
- Nao interrompa o desenvolvimento para pedir decisoes que podem ser inferidas pelo projeto.

## Antes de Modificar Arquivos

- Leia a estrutura do projeto.
- Identifique a arquitetura existente.
- Identifique componentes reutilizaveis.
- Identifique hooks existentes.
- Identifique contexts existentes.
- Identifique rotas existentes.
- Identifique o padrao visual existente.

## Implementacao

- Implemente a solucao completa e definitiva.
- Se uma funcionalidade depender de outra, implemente automaticamente.
- Se for necessario modificar outros arquivos para funcionar corretamente, faca isso automaticamente.
- Nunca crie codigo duplicado.
- Nunca crie componentes repetidos.
- Nunca crie funcoes repetidas.
- Reutilize componentes, hooks, contexts, estilos e utilitarios existentes.
- Mantenha o projeto organizado.

## Identidade Visual

E proibido modificar sem pedido explicito:

- Sidebar.
- Dashboard.
- Layout geral.
- Paleta de cores.
- Tipografia.
- Estilo dos cards.
- Botoes.
- Inputs.
- Espacamentos.
- Responsividade existente.

Toda nova tela deve parecer criada junto com o restante do sistema.

## Experiencia

O StudioFlow deve parecer um SaaS premium vendido por assinatura: organizado, produtivo, elegante, automatizado e integrado.
